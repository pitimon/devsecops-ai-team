# DevSecOps Setup

Initialize DevSecOps security scanning for the current project.

**Decision Loop**: On-the-Loop (AI proposes configuration, human approves)

## Setup Process

### 1. Detect Technology Stack

Scan the project to identify:

- **Languages**: Check for `*.py`, `*.ts`, `*.js`, `*.go`, `*.java`, `*.rb`, `*.rs`, `*.php`, `Cargo.toml`, `go.mod`, `pom.xml`, `build.gradle`, `package.json`, `requirements.txt`, `Gemfile`, `composer.json`
- **Frameworks**: Check for Next.js, Express, Django, Flask, Spring Boot, Rails, Laravel, FastAPI, Gin, etc.
- **Containers**: Check for `Dockerfile`, `docker-compose.yml`, `.dockerignore`
- **IaC**: Check for `*.tf`, `*.tfvars`, `k8s/`, `helm/`, `kustomize/`, `ansible/`, `pulumi/`
- **CI/CD**: Check for `.github/workflows/`, `.gitlab-ci.yml`, `Jenkinsfile`, `.circleci/`
- **Package managers**: npm, yarn, pnpm, pip, poetry, bundler, cargo, go modules

### 2. Recommend Scan Profile

Based on detected stack, recommend which tools to enable:

| Stack Detected      | Tools to Enable                     |
| ------------------- | ----------------------------------- |
| Any source code     | Semgrep (SAST), GitLeaks (Secrets)  |
| package.json/go.mod | Grype (SCA), Syft (SBOM)            |
| Dockerfile          | Trivy (Container)                   |
| Terraform/K8s       | Checkov (IaC)                       |
| Web app with URL    | ZAP (DAST) — requires user approval |

### 3. Check Prerequisites

Run `bash ${CLAUDE_PLUGIN_ROOT}/scripts/check-prerequisites.sh` to verify:

- Docker installed and running
- Docker Compose available
- Sufficient disk space (>2GB)
- Network connectivity for image pulls

### 4. Create Configuration

Create `.devsecops.yml` in the project root:

```yaml
# DevSecOps AI Team Configuration
# Generated by /devsecops-setup on <date>

project:
  name: <detected-project-name>
  stack:
    languages: [<detected>]
    frameworks: [<detected>]
    has_containers: <true/false>
    has_iac: <true/false>

scans:
  sast:
    enabled: true
    tool: semgrep
    rules: ["p/security-audit", "p/owasp-top-ten"]
  secret:
    enabled: true
    tool: gitleaks
  sca:
    enabled: <if-has-deps>
    tool: grype
  container:
    enabled: <if-has-docker>
    tool: trivy
  iac:
    enabled: <if-has-iac>
    tool: checkov
  dast:
    enabled: false # Requires explicit user approval
    tool: zap
    target_url: ""
  sbom:
    enabled: <if-has-deps>
    tool: syft
    format: cyclonedx-json

policy:
  role: developer
  fail_on: [CRITICAL]
  max_age_hours: 24

runner:
  mode: minimal # minimal | full
  network: devsecops
```

### 5. Install Runner (if approved)

Ask the user if they want to start the sidecar runner:

- **Minimal mode**: `docker-compose.minimal.yml` — oneshot, no persistent containers
- **Full mode**: `docker-compose.yml` — persistent sidecar with all tool profiles
